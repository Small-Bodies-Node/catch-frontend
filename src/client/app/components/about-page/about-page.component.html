CATCH Overview

The Comet-Asteroid Telescopic Catalog Hub (CATCH) is a moving-target search tool for large data sets, such as time-domain or near-Earth object surveys. You submit the name of an asteroid or comet, and CATCH will request an ephemeris for that object from NASA JPL's Horizons tool, then compute whether the ephemeris intersects any images within its database of observations. If any matches are found, CATCH displays all the salient data to you. The telescopic surveys are the haystack, and the comet/asteroid is the needle that CATCH will help you find. Use CATCH to help identify pre-discovery detections, to examine past cometary activity, or to understand why a survey did not detect or report observations of an object.

CATCH Limitations and Caveats

CATCH does not perform detection-based searches. i.e., it cannot determine if an object was observed. Rather, it identifies images that could have observed a target, based on the target's ephemeris. This search requires that the ephemeris correctly describes the target's motion on the sky. CATCH properly considers the source data observatory when computing ephemerides. CATCH does not consider ephemeris uncertainties in the search, but this capability has been reserved for a future enhancement.

Data sources

Near-Earth Asteroid Tracking (NEAT) survey data are from the NASA Planetary Data System archive of Bauer and Lawrence (2013):

  • NEAT GEODSS 1m telescope at Haleakala Observatory, April 1996 – May 1998.
  • NEAT Palomar (Tri-Cam), using the 1.2m Oschin Schmidt telscope at Palomar Observatory, November 2001 – March 2003.

SkyMapper Southern Survey data are from the Austrailian National Observatory. The telescope is located at Siding Spring Observatory. Data release 2 covers the period March 2014 – March 2018.

Known issues

  1. NEAT astrometic solutions are only approximate, and typically 10s of arcsec off.
  2. SkyMapper cutout service is not yet compatible with CATCH. Images may be downloaded with the cutout URLs provided for each search result.
  3. Target names: our name resolution service is incomplete. Some real target names may not be resolved. However, CATCH can still attempt to search for them. Ultimately, any target must be resolvable by Horizons . Use JPL's Small-Body Database Browser to test object names that are not working.
